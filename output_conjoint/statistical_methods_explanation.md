# URBANER 聯合分析——統計步驟完整說明

> **適用場合**：跨境電商競賽答辯、評審追問、團隊內部統計知識對齊
> **閱讀方式**：依「分析鏈」順序逐層閱讀，每層說明 What → Why → 本案數字 → FAQ

---

## 分析鏈總覽

```
Layer 1  資料取得（Amazon 評論 → 品質分 + 偏好標記）
         ↓
Layer 2  屬性工程（離散化 → Dummy Encoding）
         ↓
Layer 3  模型估計（Split-model Binary Logistic Regression）
              ↓ 產出：Part-worth Utilities（β 係數）
         ┌────┼──────────┬──────────┐
Layer 4a      4b        4c        4d
屬性重要性  選擇概率   WTP溢價   MNL市場份額
（Range）  （Sigmoid） （β比值） （Softmax）
         └────┴──────────┴──────────┘
                    ↓
Layer 5  Hedonic Pricing OLS（補充市場定價驗證）
```

**一句話說明整體邏輯**：用真實市場的 Amazon 評論觀察資料，透過 Logistic Regression 量化「產品屬性 → 消費者偏好」的關係，再從係數衍生出屬性重要性、願付溢價、市場競爭份額三種商業洞察。

## Layer 1：資料取得與前處理

### 做了什麼

| 項目 | 說明 |
|---|---|
| 資料來源 | Amazon 競品 SKU 評論文本 + 平均星等（avg★） |
| 屬性量化 | 使用 Axis B 品質分（0–10）代表每個 SKU 在各屬性上的品質水準 |
| Response 變數 | `preferred = 1`（若 avg★ ≥ 市場平均），否則 `preferred = 0` |
| 美國樣本 | 52 個 SKU |
| 日本樣本 | 36 個 SKU |

### 為什麼這樣做

**為什麼用 Amazon 評論而不用問卷（Survey）？**

這是 **Revealed-Preference**（顯示偏好）設計，優於問卷的 Stated-Preference（陳述偏好）：

- 評論來自真實購買後的消費者，反映實際決策行為，外部效度高
- 問卷容易有「社會期望偏誤」（受訪者說他在乎品質，其實只看價格）
- Amazon 評論的 avg★ 已聚合數十到數百位購買者的綜合評價

**為什麼用 avg★ 作為「偏好」的代理變數？**

- avg★ 是消費者對產品整體滿意度的綜合顯示
- avg★ ≥ 市場平均 → 相對受消費者偏好 → `preferred=1`
- 二元化（0/1）符合後續 Logistic Regression 的資料要求

### FAQ

> **Q：Axis B 品質分是什麼？怎麼得到的？**
> A：從評論文本做 NLP 分析，針對每個屬性（防水、附件、充電方式⋯⋯）擷取消費者的評價情緒與提及強度，量化為 0–10 分。分數越高代表市場上對該屬性的認可度越高。

> **Q：52 個 SKU 夠多嗎？**
> A：樣本偏小是本研究的已知限制。52（US）/ 36（JP）個 SKU 所有統計結果均為**方向性分析**，不宜直接外推至全市場。若要提升推論力，理想樣本數 ≥ 200 個 SKU 或搭配正式消費者問卷。

## Layer 2：屬性工程

### 做了什麼

**步驟一：離散化（將 0–10 品質分切成 3 個水準）**

| 水準 | 分數範圍 | 意義 |
|---|---|---|
| Low | 0–4 | 市場上表現較差的規格 |
| Mid | 4–7 | 市場上中等規格 |
| High | 7–10 | 市場上高規格 |

**步驟二：Dummy Encoding（虛擬變數編碼）**

每個屬性用兩個虛擬變數表示，Low 作為參照組（Reference Level）：

| 水準 | mid_dummy | high_dummy |
|---|:---:|:---:|
| Low（參照組） | 0 | 0 |
| Mid | 1 | 0 |
| High | 0 | 1 |

### 為什麼這樣做

**為什麼要離散化？**

聯合分析的核心假設是：消費者對產品的評價是由各「屬性水準」的偏好加總而來。離散化讓每個水準有獨立的係數，不強迫假設「品質分每增加 1 分，效用就線性增加」——消費者對防水等級的感受並不一定是線性的（從無防水到 IPX4 是一大跨越，從 IPX4 到 IPX7 相對較小）。

**為什麼用 Dummy Encoding 而不是 Effect Coding？**

- Dummy coding 的係數 β 直接代表「相對於 Low 水準，Mid/High 的效用增量」
- 解讀最直觀：β_mid = 從最差規格升到中等規格，消費者效用增加多少
- Effect coding 的係數是相對於「所有水準平均」，不易直接說明「升級幅度」

### 本案數字舉例

以美國市場「附件件數」屬性為例：
- Low = 附件 ≤ 3 件（mid_dummy=0, high_dummy=0）
- Mid = 附件 7 件（mid_dummy=1, high_dummy=0）
- High = 附件 ≥ 10 件（mid_dummy=0, high_dummy=1）

## Layer 3：Split-model Binary Logistic Regression（核心模型）

### 做了什麼

對**每個屬性分別**執行一條 Binary Logistic Regression：

```
logit(P(preferred=1)) = β₀ + β_mid × mid_dummy + β_high × high_dummy
```

- 因變數：`preferred`（0 或 1）
- 自變數：該屬性的 mid_dummy, high_dummy
- 輸出：**Part-worth Utilities（部分效用值）**—— β_mid, β_high

### 為什麼用 Logistic Regression 而不是 OLS？

| | OLS 線性回歸 | Binary Logistic Regression |
|---|---|---|
| 因變數要求 | 連續值 | 二元（0/1）|
| 預測結果 | 可超出 [0,1] | 自動映射至 (0,1) 機率 |
| 理論基礎 | 最小平方法 | 最大概似法（MLE）|
| 本案適用 | ❌ preferred 是 0/1 | ✅ |

Logistic Regression 估計的是：「在這個屬性水準下，產品被消費者偏好的**機率**」，完全符合離散選擇模型（McFadden, 1974）的理論框架。

### 為什麼是 Split-model（逐屬性各跑一條）而不是全屬性一起跑？

我們的資料是「真實市場 SKU」，不是實驗設計的正交卡片，屬性之間有相關性（例如：USB-C 充電的機器通常防水也好、附件也多）。若把所有屬性丟進同一條迴歸：

- 產生**多重共線性**（Multicollinearity），VIF 飆高
- 各屬性的 β 係數互相干擾，難以解讀

Split-model 每次只看一個屬性，避開共線性，係數更穩定、更可解讀。代價是無法捕捉屬性間的交互效應，但對於方向性分析已足夠。

### Part-worth Utility 是什麼意思？

β 係數在 Logit 模型中是 **log-odds**（對數勝算）單位：

- `β_mid > 0`：Mid 水準比 Low 水準更受消費者偏好（勝算比 > 1）
- `β_high > β_mid`：High 水準偏好更強
- β 的絕對大小反映：「升一個水準，偏好程度增加多少」

### 本案數字舉例

**美國市場——附件件數**

| 水準 | β 係數 | p 值 | 解讀 |
|---|---:|---:|---|
| Mid（7件 vs ≤3件） | 10.53 | 0.97 | 正向，附件越多越偏好 |
| High（≥10件 vs ≤3件） | 13.81 | 0.97 | 正向，High 偏好更強 |

> p 值偏高（接近 1.0）反映樣本小、標準誤大，係數方向可信但幅度不確定性高。

### FAQ

> **Q：p 值都很大（0.97？），這個模型有效嗎？**
> A：樣本只有 19–52 個 SKU，統計檢定力（Power）不足，導致 p 值大多不顯著。但係數的**方向**（正負）仍具有方向性參考價值——所有屬性在兩個市場的 β 方向都符合市場直覺（附件多→偏好，防水好→偏好）。正式推論需要更大樣本。

> **Q：什麼是 McFadden Pseudo R²？**
> A：Logit 模型沒有 OLS 的 R²，用 McFadden Pseudo R² 替代。計算方式：`1 - (LL_model / LL_null)`，值在 0.2–0.4 之間視為模型擬合良好。本案數值偏低，反映樣本小、噪音大。

## Layer 4a：屬性重要性（Range-based Importance）

### 做了什麼

計算每個屬性「從最差水準到最好水準，能影響效用的最大幅度」，再換算成百分比：

```
range_attr    = max(β_low, β_mid, β_high) − min(β_low, β_mid, β_high)
importance%   = range_attr / Σ(range of all attributes) × 100
```

（注意：β_low = 0，因為 Low 是參照組）

### 為什麼這樣算而不直接比較 β 大小？

- 不同屬性的 β 係數尺度不可直接比較（功能合一數的 β 達 60–70，附件件數的 β 約 13，但不代表功能合一數就「6倍重要」）
- Range-based Importance 正規化了尺度差異，讓各屬性在同一百分比尺度競爭
- 這是聯合分析領域的標準方法（Green & Srinivasan, 1990）

### 本案結果

**美國市場 Top 3**：功能合一數（51.5%）、充電方式 USB-C（10.5%）、電源類型（10.4%）
**日本市場 Top 3**：價格帶（16.9%）、附件件數（16.0%）、長度調整段數（15.5%）

### FAQ

> **Q：美國「功能合一數」重要性 51.5%，是不是太高了？**
> A：是。功能合一數在美國市場出現「完全分離」（Perfect Separation）問題——所有 preferred=1 的 SKU 功能合一分數都在某個臨界點以上，導致 β 係數異常大（|β|>60）。這是**小樣本 + 完全分離**的統計問題，而非真實消費者給功能合一數 51.5% 的重要性。在報告中此結果須附加說明，不宜直接引用於策略建議。

## Layer 4b：選擇概率（相對偏好強度）

### 做了什麼

1. 計算每個 SKU 的加總效用 U
2. Z-score 正規化
3. 透過 Sigmoid 函數轉換為 0–1 概率

```python
U_i   = Σ β_attr（各屬性 part-worth 加總）
Z_i   = (U_i − μ_U) / σ_U        # Z-score 正規化
P_i   = 1 / (1 + exp(−Z_i))      # Sigmoid 函數
```

### 為什麼要 Z-score 正規化？

若直接把 U 代入 Sigmoid：

- 美國市場功能合一數的 β 達 60–70，加總 U 可能超過 100
- Sigmoid(100) ≈ 1.0000000000（完全飽和），所有高分 SKU 的概率都無限接近 1.0
- 無法區分排名，選擇概率失去意義

Z-score 正規化後，所有 SKU 的效用在同一相對尺度競爭，Sigmoid 的輸出才有區分度。

### 「選擇概率」的正確解讀

這裡的概率**不是**「100 位消費者中有幾位會購買」的市場份額，而是：
> **「這個 SKU 相對於市場平均水準，被消費者選擇的相對偏好強度」**

P > 0.5 代表該 SKU 優於市場平均；P 越高，競爭力越強。

### 本案結果

**美國 Top 1**：ASIN `B0D2DRS9KY`，選擇概率 0.659
**日本 Top 1**：ASIN `B0742G961R`，選擇概率 0.708

## Layer 4c：WTP 願付溢價（Willingness to Pay）

### 做了什麼

用 Part-worth Utility 換算成「消費者願意為某屬性升級多付的金額」：

```
WTP（屬性升級 vs Low）= β_attr / |β_price|
```

**分子 β_attr**：該屬性從 Low 升至 Mid 或 High 的效用增量（來自 Layer 3）

**分母 |β_price|**：價格每增加 $1，效用減少多少（需補入真實售價重跑 Logit）

### 為什麼要補入真實售價？

Layer 3 的模型用「品質分（0–10）」作為屬性，不是實際貨幣金額。要把效用換算成「值多少錢」，需要：
1. 整合各 ASIN 的真實 Amazon 售價（USD / JPY）
2. 重跑一條 Logit，把「售價」也當作屬性之一
3. 估計 `β_price`（價格係數）

```python
logit(P(preferred=1)) = β₀ + β_attr × 品質分 + β_price × 實際售價
```

符合經濟理論時，`β_price < 0`（價格越高，消費者偏好越低）。

### WTP 公式背後的邏輯推導

```
效用無差異條件：
β_attr × Δ品質 + β_price × ΔWTP = 0

移項：
ΔWTP = −(β_attr / β_price) = β_attr / |β_price|
```

意思是：消費者多願意付 ΔWTP 元，剛好能抵消屬性升級帶來的效用增量——這就是「願付溢價」的貨幣化定義。

### 本案結果與限制

| 市場 | β_price | p 值 | 可靠性 |
|---|---:|---:|---|
| 美國（USD） | −0.0212 | 0.3864 | ✅ 符號正確，方向性可信，但不顯著 |
| 日本（JPY） | +0.0004 | 0.0963 | ⚠️ 符號為正，WTP 不具定價意義 |

**日本 β_price 為正的解釋**：在本案日本樣本中，高售價的 SKU 評分反而較高——這是「品質訊號效果」（Price-as-quality heuristic）。消費者把高價格當作品質保證，導致 Logit 估計出正的 β_price，違反 WTP 的計算前提。此為樣本小（21 個 SKU）且市場特殊性所致，日本 WTP 數字僅供屬性排序方向參考，不宜用於定價決策。

### 美國市場 WTP 結果（可引用）

| 屬性 | WTP（$/品質分） | p 值 | 解讀 |
|---|---:|---:|---|
| 機身尺寸 | $89.81 | 0.021 * | 從大型升至標準/迷你，消費者願意多付 |
| 附件件數 | $79.86 | 0.017 * | 每提升一個品質分，消費者多付 ~$80 |
| 長度調整段數 | $78.18 | 0.011 * | 顯著，調整段數是美國消費者真正在意的 |
| 充電方式 USB-C | $32.39 | 0.035 * | USB-C 有溢價，但幅度遠小於套組豐富度 |

### FAQ

> **Q：WTP $89.81/品質分 是什麼意思？**
> A：品質分是 0–10 的 Axis B 分數。若機身尺寸的品質分從 3 分升到 8 分（+5 分），WTP 約為 $89.81 × 5 = $449——但實際售價不可能提高這麼多，這說明樣本的 β_price 幅度估計仍有不確定性，WTP 結果只能用於**相對排序**（哪個屬性比哪個屬性更值錢），而非精確定價。

> **Q：WTP 和屬性重要性的排名不一樣，哪個更可信？**
> A：兩者問的問題不同。**屬性重要性（Range）**問的是「這個屬性對偏好的影響範圍有多大」；**WTP** 問的是「消費者願意用多少錢換取這個屬性的升級」。兩者都有參考價值，差異代表兩種不同維度的消費者決策機制。

## Layer 4d：MNL 市場偏好份額（Share-of-Preference）

### 做了什麼

把 URBANER 的 SKU、所有競品 SKU、以及 24 張虛擬設計卡片，全部丟進同一選擇集，用 Multinomial Logit（MNL）計算各產品的市場偏好份額：

```
P(選擇產品 i) = exp(U_i) / Σ exp(U_j)     j ∈ 全選擇集
```

`U_i = Σ (β_mid × mid_dummy + β_high × high_dummy)`（β 來自 Layer 3）

### MNL 和 Binary Logit 的關係

| | Binary Logit（Layer 3） | MNL（Layer 4d） |
|---|---|---|
| 選擇問題 | 買 vs 不買（2 選 1） | 從多個產品中選一個（N 選 1） |
| 數學形式 | Sigmoid：`1/(1+e^{-U})` | Softmax：`e^{U_i}/Σe^{U_j}` |
| 結果解讀 | 某產品被偏好的概率 | 在市場競爭中的份額 |
| 本質 | 相同（MNL 是 Binary Logit 的推廣） | ← 同左 |

MNL 確保所有選項的份額加總 = 1.0（100%），符合市場份額的概念。

### 為什麼用 MNL 做市場份額分析？

競賽最終問的不只是「消費者喜不喜歡這個屬性」，而是「**URBANER 在市場上能搶到多少份額？**」。MNL 提供了這個框架：

1. 把 URBANER 和競品放在同一尺度上比較
2. 計算相對於所有競品的選擇機率
3. 模擬「若 URBANER 提升某個屬性，市場份額會增加多少？」

### IIA 假設（Independence of Irrelevant Alternatives）

MNL 有一個著名限制：假設「新競品進入市場，所有現有選項的份額等比例縮減」，不允許特定產品組合之間的替代效果更強。這是 MNL 已知的理論限制，在本案的市場初步模擬階段是可接受的近似。

### 本案結果

**美國市場**

| 群組 | 偏好份額 |
|---|---:|
| URBANER 合計 | 5.2% |
| 競品合計 | 94.7% |

**日本市場**

| 群組 | 偏好份額 |
|---|---:|
| URBANER 合計 | 11.3% |
| 設計卡片 合計 | 3.1% |
| 競品合計 | 85.6% |

**設計卡片（Card 22）在日本全選擇集排名第 9**，偏好份額 3.1%，優於多數競品 SKU，代表此虛擬規格方案在日本市場具有競爭力。

### 屬性升級模擬（What-if）

「若 URBANER 將長度調整段數品質分升至 High（10 分），市場份額會如何？」

| 市場 | 現況份額 | 升級後份額 | 增幅 |
|---|---:|---:|---:|
| 日本 | 1.6% | 94.8% | **+93.2%** |

> 注意：94.8% 是理論模型的極端值，因為單一屬性升至「完美 High」會在 softmax 中產生壓倒性優勢。實際意義在於：**長度調整段數是日本市場 URBANER 最有槓桿的改善點**，而非字面上「升級後能搶到 94.8%」。

### FAQ

> **Q：MNL 和 Layer 4b 的選擇概率有什麼不同？**
> A：Layer 4b 的選擇概率（Sigmoid）是「單一產品相對市場均值的偏好強度」，沒有和競品直接競爭。MNL（Layer 4d）是「在明確選擇集中，URBANER 和每一個競品直接競爭後的市場份額」，更接近真實競爭情境。

> **Q：Card 22 是什麼？**
> A：Card 22 是由正交設計（Orthogonal Design）生成的 24 張虛擬產品規格卡片之一，用來模擬「如果 URBANER 推出這樣規格的新品，競爭力如何？」。Card 22 在日本排名第 9，代表其屬性組合在日本市場具有高競爭力。

## Layer 5：Hedonic Pricing OLS（補充市場定價驗證）

### 做了什麼

以「競品真實售價的對數值」為因變數，屬性品質分（0–10）為自變數，進行 OLS 多元線性回歸：

```
ln(Price) = β₀ + β₁×屬性1品質分 + β₂×屬性2品質分 + … + ε
```

每個 β 的解讀：**市場上的競品，該屬性品質分每提升 1 分，售價平均高出 β × 100%**（半對數模型解讀）。

### 為什麼取對數（半對數模型）？

- 售價是正值且分佈右偏（少數高價品拉高均值）
- 取對數後近似常態分佈，符合 OLS 的殘差假設
- 係數解讀從「元」變成「百分比漲幅」，更直觀

### 和 WTP 的差異

| 維度 | WTP（路線 A） | Hedonic Pricing（路線 C） |
|---|---|---|
| 問的問題 | 消費者願意多付多少？ | 市場實際如何用屬性定價？ |
| 視角 | 需求面（消費者偏好） | 供給面（市場定價行為） |
| 方法 | Logit 係數比（β_attr / β_price） | OLS 半對數回歸 |
| 用途 | 功能升級的定價決策 | 市場定位、競品定價分析 |

兩者可以互相驗證：若某屬性 WTP 高，且 Hedonic 也顯示高 β，代表消費者願意付且市場確實這樣定價，結論更可信。

### 本案限制（誠實揭露）

| 市場 | R² | Adjusted R² | 說明 |
|---|---:|---:|---|
| 美國 | 0.425 | 0.059 | Adj-R² 接近 0，模型解釋力弱 |
| 日本 | 0.272 | −0.120 | Adj-R² 為負，迴歸不穩定 |

**原因**：有效樣本僅 19（US）/ 21（JP）個 SKU，相對於 7 個自變數，自由度嚴重不足（n/p 比例 < 3）。所有係數 p 值均不顯著。

**結論**：Hedonic Pricing 在本案中**僅供定性參考**（屬性與售價的相關方向），不宜用於精確的定價回歸。此限制已在報告中誠實揭露。

## 關鍵術語速查

| 術語 | 中文 | 一句話說明 |
|---|---|---|
| Revealed-Preference | 顯示偏好 | 從真實行為（而非問卷）觀察消費者偏好 |
| Part-worth Utility | 部分效用值 | 各屬性水準對總效用的貢獻量（β 係數） |
| Split-model | 分割模型 | 逐屬性分開跑迴歸，避免多重共線性 |
| Dummy Encoding | 虛擬變數編碼 | 將類別水準轉換為 0/1 變數 |
| Range-based Importance | 範圍法重要性 | 屬性能影響效用的最大幅度，換算成百分比 |
| WTP | 願付溢價 | 消費者願意為屬性升級多付的金額 |
| MNL | 多項 Logit 模型 | 在多個產品選項中計算各產品的選擇機率 |
| IIA | 無關替代方案獨立性 | MNL 的假設：加入新選項，現有份額等比例縮減 |
| Hedonic Pricing | 享樂定價法 | 從市場售價反推各屬性的隱含價格 |
| Softmax | Softmax 函數 | MNL 的數學核心，確保所有份額加總為 1 |
| Sigmoid | Sigmoid 函數 | Binary Logit 的輸出函數，映射至 (0,1) |
