# Review Rubric And Rewrite Rules

## 1. 審稿優先順序

固定順序：

1. `Why -> What -> How -> Proof -> Ask`
2. 九段式章節完整度
3. 數字一致性
4. 常見錯誤
5. 語氣與可信度

## 2. Findings 結構

- `Critical`
- `Major`
- `Minor`

## 3. Critical

以下任一命中，直接視為不可送審：

- 沒有 Ask
- 沒有明確問題定義
- 財務數字無推導
- 把猜測寫成事實
- 缺少風險與因應
- 空洞開場導致決策者不知道問題是什麼

## 4. Major

以下屬重大缺陷：

- KPI 只有目標沒有反推
- 競品分析是名單，不是比較
- 方案只講功能，不講成果
- 行銷計畫只有通路沒有漏斗或單位經濟
- 執行計畫只有甘特圖，沒有可信度與決策點
- 問題寫成「大家都有這個問題」
- 只貼產業報告，沒有需求缺口

## 5. Minor

以下屬可修整問題：

- 語句不夠俐落
- 用字偏翻譯腔
- 視覺或標題層級不清
- 例證不足但核心邏輯已成立

## 6. `common_mistake_flags`

逐項檢查並標示 `pass|fail`：

- `generic_opening`
- `report_dumping`
- `everyone_problem`
- `kpi_without_backsolve`
- `competitor_list_only`
- `feature_without_outcome`
- `channel_without_effectiveness`
- `gantt_without_trust`
- `numbers_without_logic`
- `hidden_risks`
- `missing_ask`
- `empty_corporate_language`

## 7. Rewrite 規則

- 先修 `Critical`
- 再修 `Major`
- 最後整理 `Minor`

每個重寫建議至少包含：

- 問題句
- 原因
- 重寫方向
- 若需要，附一段重寫版本

## 8. `reintegrated_rewrite`

當原稿缺章節或缺 Ask 時，補寫時要直接整合回正文邏輯，不能只給零散備註。
