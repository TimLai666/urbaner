# Proposal Structure By Type

## 共通骨架

所有提案都用九段式：

1. 執行摘要
2. 問題定義
3. 目標客群與決策旅程
4. 競爭分析與差異化
5. 方案設計與商業模式
6. 行銷策略與推廣計畫
7. 執行計畫與團隊可信度
8. 財務預估與風險評估
9. Ask / 決策請求 / 下一步

## `internal`

### 最該強化

- 問題的組織成本
- 資源投入與回收邏輯
- 團隊執行能力
- 里程碑與 Go/No-Go

### Ask 範例

- 核准預算
- 核准 headcount
- 核准試點專案
- 核准跨部門協作

## `fundraising`

### 最該強化

- 為什麼是現在
- 市場機會與成長曲線
- 團隊可信度
- 資金用途與下一輪故事

### Ask 範例

- 本輪募資金額
- 估值區間或結構
- 資金使用規劃
- 預期里程碑

## `partnership`

### 最該強化

- 雙方可得利益
- 角色分工
- 導入流程
- 合作後的共同 KPI

### Ask 範例

- 啟動 PoC
- 簽 MOU 或合作意向
- 提供通路、資料、資源
- 指定窗口與決策時間

## 寫作提醒

- `internal` 避免寫成創業故事
- `fundraising` 避免寫成內部執行 memo
- `partnership` 必須同時說明對方加入理由
