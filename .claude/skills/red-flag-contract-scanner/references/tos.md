# 服務條款 / 隱私政策紅旗清單(Terms of Service & Privacy Policy)

涵蓋:App / 網站 / SaaS 的 Terms of Service、User Agreement、Privacy Policy、EULA、Acceptable Use Policy。

ToS 通常是「take it or leave it」,你能做的不是協商,而是搞清楚自己同意了什麼。如果太誇張,選擇不用是唯一的「協商」。

## ToS 的特殊性質(跟一般合約不同)

1. **不對稱**:對方寫好你按下同意,你沒辦法逐條改
2. **可隨時修改**:絕大多數 ToS 都保留單方修改權
3. **長且難讀**:設計上就是讓你不讀
4. **適用外國法**:美國公司用美國法、加州管轄是常態
5. **內容隨產品變動**:今天的 ToS 跟一年前可能完全不同

所以分析 ToS 時的目標不是「找出你能協商什麼」,而是:
- **告訴使用者最糟的 5-10 件事**
- **指出你放棄了哪些重要權利**
- **指出對方可以對你做什麼**
- **讓使用者決定是否值得用這個服務**

## 使用者通常最想知道的問題(用這個架構回答)

報告開頭就回答這 6 個問題,然後再進入細節:

1. **他們收集你什麼資料?**
2. **他們能拿你的資料做什麼?能賣嗎?**
3. **他們能不告知就修改條款嗎?**
4. **你放棄了哪些重要權利(訴訟權、集體訴訟、退費)?**
5. **他們能對你的內容(發文、上傳檔案、聊天紀錄)做什麼?**
6. **發生爭議時,你能怎麼辦?**

## ToS 紅旗條款

### 🚨 紅旗 #1:資料收集範圍過寬

**長什麼樣**:
- 「我們收集你提供的所有資料,以及自動收集的裝置 / 位置 / 行為資料」
- 「我們可能會收集第三方提供的關於你的資料」(從 broker 買資料)
- 「我們收集你聯絡人 / 通訊錄 / 位置歷史 / 健康資料 / 通話記錄」(看 App 性質是否合理)
- 「我們會在背景持續追蹤位置 / 行為,即使你沒在使用本服務時」

**檢查**:
- 收集的資料是否「服務必要」?例如導航 App 收位置合理,記事本 App 收位置 → 紅旗
- 是否區分「必要 vs 可選」?可選資料是否能拒絕?
- 拒絕特定資料收集是否會影響服務(可以,但要合理)

### 🚨 紅旗 #2:資料分享 / 販售

**長什麼樣**:
- 「我們可能與業務夥伴 / 廣告商 / 第三方分享你的資料」
- 「我們可能將去識別化資料用於商業用途」(去識別化標準很模糊)
- 「在合併、收購或破產情況下,你的資料可能被轉移」
- 「我們不販售你的資料」 + 但「分享給合作夥伴用於行銷」(實質就是賣)

**檢查**:
- 「不賣」跟「分享」的差別 —— 很多公司宣稱不賣但分享,實質一樣
- 與第三方分享的「目的」是否限縮?(寫「為提供更好服務」等同於沒寫)
- 第三方名單是否揭露?
- 你能否退出資料分享?(opt-out)
- 是否符合台灣個資法 + GDPR(若服務歐盟使用者)

**台灣個資法重點**:
- 蒐集 / 處理 / 利用須有特定目的 + 告知 + 同意
- 你有「查詢、更正、刪除、停止蒐集」之權利
- 違反 → 可向個資保護機關 / 法院主張賠償

### 🚨 紅旗 #3:你內容的授權範圍

**長什麼樣**(社群平台 / 雲端 / 創作平台特別注意):
- 「使用者授予本平台全球性、永久、可再授權、可修改、免費之使用權」
- 「本平台得以任何形式使用、改作、散布、公開傳輸使用者上傳之內容」
- 「使用者保證對上傳內容擁有完整權利,並對本平台之使用承擔全責」

**為何紅旗**:
- 「永久」 → 你刪了帳號,他們還能用
- 「可再授權」 → 他們可以授權給其他公司
- 「可修改」 → 你的作品可能被改頭換面後使用
- 「免費」 → 你的內容變成他們的素材,沒分潤

**正常範圍**:
- 限縮在「為提供服務必要之範圍」
- 你刪除內容後授權應終止
- 不可再授權第三方
- 不能用於非服務目的(例如行銷)

**檢查重點**:你發在這個平台的東西,他們能不能拿去訓練 AI、能不能放到別的廣告、能不能轉賣?

### 🚨 紅旗 #4:單方修改權 + 通知方式弱

**長什麼樣**:
- 「我們得隨時修改本條款,修改後公布於網站即生效」
- 「使用者於修改後繼續使用即視為同意新條款」
- 「我們可能透過電子郵件 / App 通知,但不擔保送達」

**為何紅旗**:你今天簽的條款,明天可能變成完全不同的。對方靠「繼續使用 = 同意」綁架你。

**檢查**:
- 重大變更是否有「事前通知 + 你可拒絕並終止服務」?
- 變更的「重大性」由誰判斷?
- 修改後的條款是否能查到歷史版本?

### 🚨 紅旗 #5:強制仲裁 + 放棄集體訴訟

**長什麼樣**(美國公司常見):
- 「Any disputes arising out of this agreement shall be resolved by binding arbitration」
- 「You waive your right to participate in any class action」
- 「Disputes shall be resolved individually」

**為何紅旗**:
- 仲裁 → 沒有上訴機會、費用高、結果不公開
- 放棄集體訴訟 → 你個人吃虧不值得告,但 100 萬人吃虧加起來告就有意義 → 他們最怕這個

**在台灣**:消費者保護法對消費爭議有「不得限制提起訴訟」之保護,但跨境服務常以「適用美國法」迴避。

### 🚨 紅旗 #6:服務終止 / 帳號停權

**長什麼樣**:
- 「我們得隨時、無需理由終止你的帳號」
- 「終止後,你已支付之費用恕不退還」
- 「終止後,你的資料 / 內容 / 點數 / 虛擬貨幣 / 訂閱權益均喪失」
- 「我們對終止造成之損害概不負責」

**為何紅旗**:你買的東西、你累積的內容,可能在他們一個按鈕後消失。

**檢查**:
- 終止是否需要「合理原因」?
- 終止後資料 / 內容是否能匯出?匯出期限多久?
- 已付費部分是否退費?
- 是否有「申訴 / 復原」機制?

### 🚨 紅旗 #7:責任免除 + 賠償上限離譜

**長什麼樣**:
- 「服務以現狀提供,不擔保適合性、可用性、不中斷、無錯誤、無病毒、無資料遺失」
- 「對任何直接、間接、附帶、特別、衍生性損害概不負責」
- 「總賠償責任上限為 [過去 12 個月你支付給我們的費用 / 100 美元 / 0 元]」

**為何紅旗**:雲端服務當機 → 你損失幾百萬資料 → 他們最多賠 100 美元。

**檢查**:
- 賠償上限的金額是多少?
- 是否包含「故意 / 重大過失」?(在台灣這類免責通常無效,但仍會在訴訟上被引用)
- 你的核心使用情境是否得到保障(例如:雲端儲存對「資料遺失」應有實質保障)

### 🚨 紅旗 #8:司法管轄 / 適用法律

**長什麼樣**:
- 「本協議適用美國加州法,爭議由舊金山法院管轄」
- 「本協議適用開曼群島法 / 香港法 / 新加坡法」

**為何紅旗**:你不可能跨國打官司。

**檢查**:對台灣消費者而言,這類條款雖然台灣消保法可能不認,但在實務上你還是很難主張。

### ⚠️ 注意 #1:Cookie / 追蹤技術

**檢查**:
- 是否使用第三方追蹤(Google Analytics、Meta Pixel、廣告網絡)
- 你是否能拒絕(opt-out)?拒絕後是否影響使用?
- 是否符合 GDPR 「明示同意」(若服務歐盟使用者)

### ⚠️ 注意 #2:跨境資料傳輸

**長什麼樣**:「資料可能儲存於 [美國 / 中國 / 其他國家] 伺服器」

**檢查**:
- 對你的個資保護是否充分?
- 該國政府是否有寬泛資料調取權(美國 CLOUD Act、中國資料安全法)?
- 你是否在意資料出境?

### ⚠️ 注意 #3:未成年使用者

**檢查**:
- 服務是否限制 [13/16/18] 歲以下使用?
- 對於未成年使用者的資料保護是否有額外規定?(美國 COPPA、台灣個資法兒少保護)

### ⚠️ 注意 #4:AI 訓練條款

**長什麼樣**(2023 後常見):
- 「我們可能使用你的內容訓練 AI 模型」
- 「使用我們的 AI 服務時,你的輸入 / 輸出可能用於改進服務」

**檢查**:
- 是否能 opt-out?
- 企業 / 付費版本是否有不同政策?
- 你輸入的敏感資訊(商業機密、個資)是否會被用來訓練?

### ✅ 通常正常的條款

- 帳號安全(密碼自行保管、不得共用)
- 禁止違法使用(發布違法內容、騷擾、詐騙)
- 智慧財產權標示(平台 logo、設計屬於平台)
- 第三方連結免責(連到外部網站不為其內容負責)
- 不可抗力豁免(範圍合理)
- 條款可分性(部分無效不影響其他)

## ToS 分析的特殊輸出格式

對 ToS 類文件,在 SKILL.md 主流程的基礎上,額外加上:

### 📋 「你同意中最糟的 5 件事」 TOP 5 摘要

放在報告最前面,直接告訴使用者最該記得的 5 件事:

```markdown
## 📋 你同意中最糟的 5 件事

1. **[簡短名稱]**:[一句話講清楚]
2. **[簡短名稱]**:...
3. **[簡短名稱]**:...
4. **[簡短名稱]**:...
5. **[簡短名稱]**:...
```

### 📋 6 大問題快答

```markdown
## 📋 你最想知道的問題

| 問題 | 答案 |
|------|------|
| 收集什麼資料? | [具體列出] |
| 能賣 / 分享嗎? | [是/否 + 給誰] |
| 能不告知就改條款嗎? | [是/否 + 怎麼通知] |
| 你放棄了哪些權利? | [集體訴訟? 訴訟管轄? 退費?] |
| 你的內容他們能怎麼用? | [授權範圍] |
| 出問題能怎麼辦? | [仲裁/訴訟/賠償上限] |
```

## 提醒使用者的事

1. **ToS 沒得協商,但有得選擇**:看完不能接受 → 不用這個服務
2. **企業 / Pro 版本通常條款不同**:免費版被拿資料,付費版可能不會,要看 Enterprise Agreement
3. **個資法是你的盾牌**:即使 ToS 寫了不利條款,你仍有「查詢、更正、刪除」之權利
4. **重大事件先存證**:如果你發現服務違反自己的隱私政策(例如沒公告就改條款),截圖留證 → 後續申訴有用
5. **GDPR / CCPA 有效**:你不是歐盟 / 加州使用者也可主張(很多公司全球套用)
