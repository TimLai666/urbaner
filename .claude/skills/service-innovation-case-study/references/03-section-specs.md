# 各報告區段的詳細規格與品質標準

## 課堂報告必要區段（對照課堂投影片）

以下是完整的課堂格式要求，對應投影片中的「涵蓋內容」清單：

1. **案例名稱**：公司全名 + 業務一句話描述
2. **推出時間**：時間線（不只是一個年份）
3. **推薦原因**：包含真實口碑（具名引言）
4. **應用情境**：B2C + B2B + 創辦人願景，分層說明
5. **創新類型**（六選一）
6. **創新模式**（四項，複選）
7. **鎖定客群**（STP）
8. **服務模式 / 商業模式**（BMC）
9. **創造效果 / 解決問題**（三層分析）
10. **相關展示**（影片或實際使用，非必要）
11. **目前成效 / 獲獎情況 / 外部討論關鍵**
12. **背後代表啟發或意義**
13. **參考資料 / 下載連結**

---

## 各區段詳細規格

### 推出時間（時間線）

**格式**：每個節點一個段落，節點後面緊接來源標注
**必須包含的節點類型**：
- 創辦人相識 / 公司正式成立
- 獲得首輪投資
- 進入 stealth / beta 測試
- 第一波媒體曝光
- App Store / 平台正式上線
- 宣布重要融資
- 重要功能更新
- 獲獎或行業認可

**品質標準**：
- 每個節點都要有明確的時間（「2025 年 X 月」，而非「2025 年」）
- 每個節點都要有來源標注（用 `> 來源：` 格式）
- 若來源有衝突，說明採信哪個並解釋原因

---

### 推薦原因（含真實口碑）

**必須包含**：
- 服務概述（1 段）
- 至少 3-5 則具名引言，涵蓋不同類型的聲音：
  - 投資人（代表資金認可）
  - 媒體評測（代表第三方客觀評估）
  - 具名用戶（代表真實使用體驗）
  - 如有：名人、KOL 的背書

**每則引言必須有**：
- 說話者姓名
- 說話者身份
- 原文引言（翻譯可以，但附原文更好）
- 出處（媒體名稱 + 日期）

---

### 應用情境

**三層結構**：
1. 消費者端（B2C）：功能逐一列出，每個功能附來源
2. 品牌端（B2B）：商業合作模式、API 或技術授權方式
3. 創辦人未來願景：訪談或公開聲明中提及的規劃方向

---

### 創新類型（六選一）

**六個選項**：
- 核心服務創新（全新服務）
- 附屬創新服務
- 服務流程創新
- 既有服務流程延伸
- 相關服務和產品的延伸
- 服務風格改變

**選擇標準**：
- 選一個最貼切的，並說明為何選這個而非其他
- 排除其他選項的理由也要說明
- 必須引用創辦人的一句話作為佐證

---

### 創新模式（複選）

**四個選項**：
- 新服務概念
- 新客戶服務介面
- 新服務傳遞方式
- 新科技應用

**選擇標準**：
- 可複選（通常選 2-4 個）
- 每個選中的選項要說明具體的體現方式
- 每個說明要有事實依據

---

### 目前成效 / 獲獎情況 / 外部討論

**結構**：
1. 可查證數字表格（融資、員工數、覆蓋市場等）
2. 獲獎與外部認可（有來源）
3. 媒體報導時間軸（表格）
4. 關鍵外部聲音（正面 + 批評並列）

**絕對不能出現**：
- 未查證的用戶數字（「擁有百萬用戶」等無來源數字）
- 未查證的成長率
- 「廣受好評」等沒有具體引言的模糊描述

---

### 背後代表的啟發與意義

**格式**：
- 每個啟發一個標題段落
- 每個啟發第一句說明分析依據來源（「從 PESTEL T-01 優先分 75 可以推導出...」）
- 每個啟發提出一個可以應用到其他案例的普遍規律

**標準啟發結構**：
1. **[啟發名稱]**
   分析依據：[來自哪個分析模型的什麼結論]
   具體體現：[在這個案例中如何體現]
   普遍規律：[可以應用到其他案例的通則]

---

## 公司背景 + 創辦人背景（補充規格）

### 公司背景必須包含：
- 成立年份、地點、全名
- 核心業務一句話描述
- 獲得的資金（所有輪次）
- 重要里程碑（上線時間、用戶規模——若已公開）

### 創辦人背景必須包含每位創辦人：
- 學歷（學校、科系）
- 前雇主完整列表（不只是最有名的那個）
- 在前雇主的具體職位與工作內容
- 為何這個背景與現在的公司相關

### 早期輪投資背景（若有）必須包含：
- 投資方的詳細介紹（不只是名稱）
- 投資方的投資理念或 LP 結構
- 「如何被發現」的具體故事
- 投資理由（投資人的公開聲明）

---

## 參考資料區段規格

**格式**：
```
## 參考資料

### 一、主要媒體與報導
1. **[媒體名]**（[日期]）— [標題或說明]
   [URL]

### 二、產業分析與評測
...

### 三、投資方與融資資料
...

### 四、創辦人與公司一手來源
...

### 五、數據與市場研究
...
```

**規則**：
- 每個來源只出現一次（不重複）
- URL 盡量提供（若無法提供，說明資料來源）
- 分類清楚，不要所有來源混在一起
- **絕對是最後一個區段，不能有任何內容在參考資料之後**
