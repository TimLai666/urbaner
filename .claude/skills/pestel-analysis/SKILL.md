# PESTEL Analysis Skill

本 skill 執行嚴謹的 PESTEL 總體環境分析，核心原則：**每個因素都必須有來源依據、重要性評分、影響程度評分，不得憑空生成。**

---

## 核心原則

- **有據可查**：每個因素必須能指出來源（數據、報告、法規、趨勢報導等）
- **評分才算數**：未經評分的因素不得進入最終清單
- **影響必須具體**：不能只說「有影響」，要說對誰、如何影響、影響幅度
- **雙向判斷**：每個因素都需判斷是機會傾向還是威脅傾向（或中性）
- **篩選而非羅列**：最終只保留高重要性或高影響的因素，其餘降為觀察項

---

## 分析標的確認（執行前必填）

在開始分析前，先確認以下資訊：

```
分析標的：____（公司 / 產品 / 事業單位 / 市場進入計畫）
產業別：____
地理範圍：____（台灣 / 東南亞 / 全球 / 特定市場）
分析目的：____（SWOT前置 / 市場進入評估 / 策略規劃 / 投資評估）

```

若使用者未提供，主動詢問後再執行。

## 六大面向掃描框架

### P — Political 政治法規

**掃描問題**：
- 現行法規對該產業有哪些直接限制或要求？
- 政府政策方向（補貼、管制、開放）近期有何變化？
- 政治穩定度、選舉周期、政黨立場對產業的影響？
- 貿易政策、關稅、跨境限制？
- 地緣政治風險（供應鏈、市場准入）？

**常見有效因素範例**：法規鬆綁、政府補貼計畫、進口關稅調整、資料隱私立法、外資持股限制

---

### E — Economic 經濟

**掃描問題**：
- GDP 成長率、景氣循環位置？
- 通膨率、利率走向對消費力與融資成本的影響？
- 匯率波動對原料採購、出口競爭力的影響？
- 失業率、薪資水準、消費信心？
- 產業資本投入趨勢？

**常見有效因素範例**：升息環境壓縮消費、弱勢台幣利出口、原物料成本上漲、景氣衰退降低非必要支出

---

### S — Social 社會

**掃描問題**：
- 人口結構變化（高齡化、少子化、都市化）？
- 消費習慣、生活型態轉變？
- 價值觀變化（健康意識、永續消費、個人化）？
- 勞動力供給結構？
- 文化因素對產品接受度的影響？

**常見有效因素範例**：高齡化擴大醫療需求、Z世代偏好訂閱制、健康意識提升帶動機能食品、遠距工作常態化

---

### T — Technological 技術

**掃描問題**：
- 關鍵技術是否出現顛覆性突破？
- 數位轉型、自動化對產業結構的衝擊？
- AI、大數據、IoT 的應用機會或替代威脅？
- 技術取得門檻、專利佈局？
- 供應鏈技術升級或斷鏈風險？

**常見有效因素範例**：生成式AI降低內容生產成本、電動車電池技術突破、雲端化壓低IT建置門檻、自動化取代勞動力

---

### E — Environmental 環境生態

**掃描問題**：
- 碳排放法規、ESG 要求對營運成本的影響？
- 氣候變遷對原料供給、生產地點的風險？
- 消費者對環境友善產品的偏好強度？
- 廢棄物、包裝、供應鏈碳足跡要求？
- 自然災害、極端氣候的營運風險？

**常見有效因素範例**：碳費上路增加製造成本、ESG評級影響融資條件、消費者綠色偏好擴大市場、極端氣候影響農業原料供給

---

### L — Legal 法律

**掃描問題**：
- 勞動法規（最低工資、工時、外勞）？
- 消費者保護法、產品責任？
- 智慧財產權保護強度？
- 資料保護、個資法（GDPR等）？
- 反壟斷、競爭法規？
- 產業特定執照、認證要求？

**常見有效因素範例**：個資法強化增加合規成本、最低工資調漲、電商平台責任法規收緊、醫療器材 FDA 認證門檻

---

## 評分系統

### 每個因素必須完成三項評分

**① 重要性（Importance）：此因素對該產業/公司的結構性重要程度**

| 分數 | 標準 |
|------|------|
| 1 | 邊緣影響，可忽略 |
| 2 | 有一定影響但非核心 |
| 3 | 中等重要，需納入規劃 |
| 4 | 高度重要，影響競爭格局 |
| 5 | 決定性重要，不回應將影響生存 |

**② 影響程度（Impact）：若此因素發生變化，對公司的實際衝擊幅度**

| 分數 | 標準 |
|------|------|
| 1 | 幾乎無感 |
| 2 | 小幅影響（<5% 營收/成本） |
| 3 | 中等影響（5-15%） |
| 4 | 顯著影響（15-30%） |
| 5 | 重大衝擊（>30% 或結構性改變） |

**③ 時效性（Urgency）：此因素在分析時間視窗內的緊迫程度**

| 分數 | 標準 |
|------|------|
| 1 | 遠期（3年以上才具體化） |
| 2 | 中期（1-3年內） |
| 3 | 近期（12個月內將明顯影響） |

### 綜合優先分

```
優先分 = 重要性 × 影響程度 × 時效性

最高分：5 × 5 × 3 = 75
篩選門檻：優先分 ≥ 12 → 列為有效因素
           優先分 6-11 → 降為觀察項
           優先分 ≤ 5  → 暫時排除
```

## 因素記錄格式（強制）

每個因素必須用以下格式記錄，不得省略任何欄位：

```
[面向代碼-編號] 因素名稱
描述：具體說明此因素的內容與現況
來源依據：____（數據來源、法規名稱、報告、新聞等）
影響方向：機會傾向 / 威脅傾向 / 中性（視公司能力而定）
影響對象：____（成本結構 / 市場規模 / 競爭格局 / 客戶行為 / 供應鏈）
重要性：__/5
影響程度：__/5
時效性：__/3
優先分：__（重要性 × 影響程度 × 時效性）
列為：有效因素 / 觀察項 / 排除
```

**範例**：
```
[P-01] 再生能源發展條例修正
描述：2024年修正案要求製造業用電大戶須採購20%綠電，預計2026年上路
來源依據：經濟部能源署公告、工業局說明會資料
影響方向：威脅傾向（增加合規成本）
影響對象：成本結構、供應鏈
重要性：4/5
影響程度：3/5
時效性：3/3
優先分：36
列為：有效因素
```

## 執行流程

### 步驟一：逐面向掃描
依 P → E → S → T → E → L 順序，針對分析標的逐一掃描，每個面向至少找出 3-5 個候選因素。

### 步驟二：逐一評分
對每個候選因素完成三項評分，計算優先分。

### 步驟三：篩選分類
- 優先分 ≥ 12：列為有效因素，進入最終清單
- 優先分 6-11：降為觀察項，附帶觀察觸發條件
- 優先分 ≤ 5：排除，簡述排除理由

### 步驟四：影響方向歸類
對所有有效因素，標記影響方向：
- **機會傾向**（O候選）→ 直接輸出給 SWOT 的機會池
- **威脅傾向**（T候選）→ 直接輸出給 SWOT 的威脅池
- **中性**：依企業能力決定，需附條件說明

### 步驟五：輸出總覽表

### 有效因素清單（優先分 ≥ 12）

逐條列出，使用完整記錄格式。

### 觀察項清單（優先分 6-11）

逐條列出，附觸發條件：「當____發生時，升級為有效因素」

- [ ] 所有因素都完成三項評分且優先分計算正確
