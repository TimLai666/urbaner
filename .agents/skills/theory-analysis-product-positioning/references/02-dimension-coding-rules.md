# Dimension Coding Rules

## Theory Link

完整理論概念、構面意涵與研究應用請先讀：

- `references/00-theory-foundation.md`

本文件只保留標註時需要的 operational 規則與判準，避免與理論章節重複。

## Allowed Taxonomy

- `family`: `product_positioning`
- `subtheory`:
  - `attributes`
  - `functions`
  - `benefits`
  - `usage_context_service_experience`

## Dimension Definitions

- `attributes`:
  - 標註焦點是外在與結構訊號（如品質、尺寸、材質、描述一致性）。
- `functions`:
  - 標註焦點是功能與效能表現（如性能輸出、穩定度、故障情況）。
- `benefits`:
  - 標註焦點是使用者價值與結果（如性價比、信任感、再購傾向）。
- `usage_context_service_experience`:
  - 標註焦點是使用流程與服務情境（如安裝、配送、售後、問題處理）。

## Coding Procedure

1. 逐項證據切分關鍵語句。
2. 每句可標註 0-2 個 subtheory，避免過度多標。
3. 每筆標註都要綁定原文 quote 與簡短理由。
4. 不確定時標記 `insufficient`，不要補腦。

## Evidence Rules

- `quote` 必須是證據內容中的連續片段。
- `reason` 需指出為何對應該構面，不可只重述原句。
- 正負向可標註為：`positive | neutral | negative`。
