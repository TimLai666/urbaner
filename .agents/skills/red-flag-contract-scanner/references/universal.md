# 通用紅旗清單(Universal Red Flags)

任何契約都要掃這 12 類。順序不代表嚴重程度,要看具體條文。

## 1. 自動續約陷阱(Auto-Renewal Traps)

**長什麼樣**:
- 「合約到期後若未於 [X] 日前以 [特定方式] 通知,視為自動續約 [Y] 期」
- 「續約期限與原約期相同/雙方議定/由甲方決定」
- 通知方式刻意難:必須掛號信、必須親自到場、必須在特定窗口

**為何是紅旗**:你忘了 → 自動再綁一年。常見於健身房、訂閱服務、租賃。

**檢查重點**:
- 通知期限長度合理嗎(30 天合理,90 天偏苛,180 天紅旗)
- 通知方式是否刁難(只接受紙本掛號 = 紅旗)
- 續約後可否漲價、能否再次取消
- 有沒有「自動續約前 X 天必須主動提醒消費者」的條款(在台灣消保法對長期服務有要求)

## 2. 隱藏費用(Hidden Fees)

**長什麼樣**:
- 主文寫月費 X 元,附件或小字寫「另收行政費 / 設施維護費 / 系統費 / 帳務處理費」
- 「甲方有權於必要時徵收合理費用」(沒定義什麼叫合理)
- 升級費、取消費、變更方案費、列印帳單費、紙本帳單費
- 第三方代收費(刷卡手續費、銀行費)由你承擔

**檢查重點**:加總所有費用,跟廣告/業務口頭講的金額對照。差距 > 10% 就是紅旗。

## 3. 違約金不對等(Asymmetric Penalties)

**長什麼樣**:
- 你違約罰高(例如剩餘期間全額 + 額外賠償)
- 對方違約罰低(只退已付金額,不賠你損失)
- 「乙方提前終止應賠償所有剩餘月費,且不得退款」
- 沒有「對方未履約時你可以怎麼辦」的條款

**檢查重點**:把「我違約 → 我要付什麼」和「對方違約 → 我能拿到什麼」並排比較。對等才正常。

## 4. 責任免除(Limitation of Liability)

**長什麼樣**:
- 「甲方對任何直接、間接、衍生性損害概不負責」
- 「服務以現狀提供,不擔保適合性、可用性、準確性」
- 「如有故障/中斷/資料遺失,甲方至多賠償 [一個月費用 / 0 元]」
- 不可抗力範圍寫得很寬(把「系統升級」「合作廠商問題」也算進去)

**為何是紅旗**:對方賣你東西,卻不為東西的品質負責。常見於 SaaS、雲端服務、ToS。

**檢查重點**:賠償上限是否離譜(賠你一個月費用,但你損失幾百萬資料 → 紅旗);免責範圍是否包含「對方故意或重大過失」(在台灣這種免責通常無效,但寫了還是想規避)。

## 5. 單方修改權(Unilateral Modification)

**長什麼樣**:
- 「甲方有權隨時修改本約條款,修改後公告於網站即生效」
- 「費率/服務內容得由甲方依市場狀況調整」
- 「甲方得依需要新增、變更、終止本服務之全部或一部」

**為何是紅旗**:今天你簽的合約,明天可能完全不一樣,而你不能反對。

**檢查重點**:有沒有「修改前需通知 + 你有拒絕權 + 拒絕可無條件終止」的保護?沒有就是紅旗。

## 6. 爭議解決條款(Dispute Resolution)

**長什麼樣**:
- 「爭議由 [遠地] 法院管轄」(台北人簽的合約寫高雄管轄 → 提告成本高)
- 「應提付 [特定機構] 仲裁,不得提起訴訟」(仲裁費用高、上訴有限)
- 適用 [外國] 法律(在台灣的合約寫適用美國法 → 紅旗)
- 限制集體訴訟(class action waiver)

**檢查重點**:管轄法院在不在你住的地方;有沒有強迫仲裁;放棄哪些訴訟權利。

## 7. 智慧財產權歸屬(IP Assignment)

**長什麼樣**(在勞動契約跟服務契約特別常見):
- 「乙方在職期間之所有發明、創作、著作均歸甲方所有」(包括下班時間做的東西)
- 「乙方之既有作品於本約生效後授權甲方無限期、無償、可再授權使用」
- 服務條款裡:「使用者上傳之內容,授予本平台全球性、永久、免費、可再授權之使用權」

**檢查重點**:是否限縮在「業務相關」「上班時間」「使用公司資源」?有沒有列出你個人既有作品的排除清單?

## 8. 競業條款 / 保密條款(Non-Compete & NDA)

**長什麼樣**:
- 「乙方離職後 [X 年] 內不得從事 [Y 範圍] 之業務」
- 沒有合理補償(在台灣勞基法第 9-1 條規定要有合理補償)
- 範圍寫得很寬(「相關產業」「同業」)
- 違反罰款很高(離職後做同行 → 賠 100 萬)

**檢查重點**:時間 / 地理 / 業務範圍 / 補償金 / 違約金 —— 五個都要看。在台灣一年內 + 合理補償 + 明確範圍才比較站得住。

## 9. 保證與擔保(Warranties & Guarantees)

**長什麼樣**:
- 你保證的事項超多(「乙方保證所提供之資料絕無錯誤、無侵權、無病毒…」)
- 對方保證的事項少到接近沒有
- 你的保證若違反 → 賠償 + 終止 + 訴追;對方違反保證 → 沒寫

**檢查重點**:雙方保證條款的數量跟嚴重程度是否對稱。

## 10. 終止條款(Termination)

**長什麼樣**:
- 對方可隨時終止,不需要理由(at-will)
- 你要終止需要嚴格條件 + 提前通知 + 賠償
- 終止後條款仍有效(post-termination obligations)寫得很長很苛
- 終止後保密 / 競業 / 不挖角條款延續多年

**檢查重點**:雙方終止權是否對等;終止後你還要扛多少義務。

## 11. 通知與聯絡方式(Notice Provisions)

**長什麼樣**:
- 「所有通知以掛號信寄送至 [甲方指定地址] 為準」(只接受紙本)
- 「以甲方系統紀錄為準」(對方紀錄勝過你的紀錄)
- 你的通知地址若變更需提前 X 天書面通知,對方變更不用

**為何是紅旗**:看似行政條款,但決定了爭議發生時誰有舉證優勢。

## 12. 整合條款 / 口頭承諾失效(Entire Agreement Clause)

**長什麼樣**:
- 「本約已涵蓋雙方所有約定,任何先前之口頭或書面承諾均不生效」
- 「除以書面修訂外,本約之任何修改均無效」

**為何是紅旗**:業務跟你講的「保證」「沒問題啦」「之後可以調整」全部失效。簽下去 = 只有紙上寫的算數。

**檢查重點**:業務有沒有口頭承諾過什麼?如果有 → 在簽約前要求把這些寫進合約附件,否則簽了就作廢。

---

## 掃描順序建議

1. 先翻最後幾頁(整合條款、爭議解決、終止 —— 最容易藏雷)
2. 再翻費用相關章節(隱藏費用、違約金)
3. 再翻雙方權利義務章節(責任免除、保證、IP)
4. 最後讀全文確認沒漏

## 紅綠燈速判

看到一條條款時,問自己這三個問題:
1. 「對方違約時我能怎麼辦?」如果沒答案 → 紅旗
2. 「這條對方為什麼要寫進來?」如果想到的理由是「為了凹你」→ 紅旗
3. 「萬一我跟對方鬧翻,這條會讓我陷入什麼處境?」如果想得出最壞情境 → 至少是注意條款

## 不是紅旗的「正常條款」舉例(別嚇使用者)

- 「雙方同意以中華民國法律為準據法」(在台灣簽的合約 → 正常)
- 「本約一式兩份,雙方各執一份」(行政條款,正常)
- 「條文若有部分無效,其他部分仍然有效」(separability,正常)
- 「不可抗力(天災、戰爭、疫情)豁免責任」(範圍合理就正常)
- 標準保密義務(雙方都負保密義務 + 範圍合理)→ 正常
- 「履行本約期間應遵守相關法令」→ 正常,本來就要

把這些標出來、註明「正常」,使用者才不會看到紅旗就以為整份合約都壞掉。
