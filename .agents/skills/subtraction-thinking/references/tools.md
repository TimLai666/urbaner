# 減法思維工具參考

## 1. 移除評估表

用途：讓「要不要拿掉這個東西」從感覺題變成可以討論的問題。

| 項目 | 存在的理由是什麼？ | 拿掉的真實損失是什麼？ | 有沒有更輕量的替代？ | 建議 |
|------|-------------------|----------------------|---------------------|------|
|      |                   |                      |                     | 移除 / 簡化 / 保留 |

**判讀**：說不清楚存在理由、或損失很模糊的，優先考慮移除或簡化。

---

## 2. 路徑分析（簡化用）

用途：找出流程、服務、決策路徑中不必要的彎路。

步驟：
1. 畫出現有路徑的每個節點
2. 對每個節點問：「這個節點對核心目的有什麼直接貢獻？」
3. 貢獻不清楚的 → 候選合併或移除
4. 是在繞過某個根本問題的 → 先修根本問題

| 節點 | 對核心目的的貢獻 | 是否可合併或移除？ | 備註 |
|------|----------------|-------------------|------|
|      |                |                   |      |

---

## 3. 優先排序（Must / Should / Could / Won't）

用途：把「這次不做」正式寫進計畫，而不是說「之後再說」。

| 項目 | Must（不做就壞） | Should（有價值但可延後） | Could（加分但非必要） | Won't（這次不做） | 理由 |
|------|-----------------|------------------------|----------------------|-------------------|------|
|      |                 |                        |                      |                   |      |

**規則**：Won't 欄一定要填，不能空著。「之後再說」要寫成「在什麼條件下才做」。

---

## 4. 選項評估表（含「不做」與「簡化現有」）

用途：避免假兩難，確保減法選項被真正評估過。

| 選項 | 對目標的貢獻 | 成本與代價 | 風險 | 總評 |
|------|-------------|-----------|------|------|
| 方案 A（新增）|  |  |  |  |
| 方案 B（簡化現有）|  |  |  |  |
| 不做 |  |  |  |  |

---

## 5. 假設驗證卡

用途：在全力投入之前，用最小成本先確認方向是對的。

```
目標對象：
我們認為存在的問題或需求：
我們的主張：
最小驗證方式：（能用多便宜的方式得到答案？）
成功的樣子是什麼？（量化）
如果假設不成立，要調整哪些方向？
```

---

## 6. 覆盤必答題

1. 這次做了哪些東西？各自對目標的貢獻是什麼？
2. 哪些東西其實可以延後或不做？當初為什麼沒有拿掉？
3. 哪裡變得比預期複雜？下次要怎麼避免？
4. 如果重來一次，第一個要刪的是什麼？
5. 現在有沒有什麼東西，已經沒有人真正在用或真正需要，但還留著？

---

## 刪減決策閾值

| 條件 | 建議處置 |
|------|---------|
| 說不清楚存在理由，或理由已經不再成立 | 優先移除 |
| 有存在理由，但有更輕量的替代方式 | 簡化或替換 |
| 有存在理由，但現在驗證不了它是否有效 | 先延後，設定觸發條件 |
| 有存在理由，但加入後整體更難理解 | 重新設計，讓它更直接 |
| 各方有爭議 | 用最小成本驗證，不用辯論解決 |
