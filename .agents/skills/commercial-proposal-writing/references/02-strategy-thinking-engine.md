# Strategy Thinking Engine

## 原則

思考占比高於落筆。企劃寫得快沒有價值，方向錯誤才是最大風險。

## 企劃五問

### 1. 為誰解決什麼問題

- 受影響的是誰
- 具體痛點是什麼
- 不行動的代價是什麼

### 2. 為什麼是現在

- 市場、政策、技術、組織時機是什麼
- 如果現在不做，會失去什麼

### 3. 憑什麼是我們

- 團隊過往成績
- 關鍵能力
- 資源優勢

### 4. 要花多少、賺多少

- 需要什麼資源
- 收益如何推導
- 回收期與單位經濟是否成立

### 5. 成功長什麼樣

- 成功指標
- 階段性里程碑
- 何時判定繼續或停止

## `strategy_answers` 輸出格式

- `for_whom_and_what_problem`
- `why_now`
- `why_us`
- `economics`
- `success_definition`

## 使用規則

1. 五問必須先於正文。
2. 若五問有任一題回答模糊，補足資料後再寫正文。
3. 五問的答案必須能映射到 `Why -> What -> How -> Proof -> Ask`。
