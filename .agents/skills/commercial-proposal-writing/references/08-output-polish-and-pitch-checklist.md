# Output Polish And Pitch Checklist

## 1. 最終健檢

- [ ] 已通過 Data Sufficiency Gate
- [ ] `Why -> What -> How -> Proof -> Ask` 完整
- [ ] 九段式章節完整
- [ ] Ask 具體且可執行
- [ ] 財務、KPI、時程前後一致
- [ ] 所有假設都有 `Assumption / Validation Needed / Risk`

## 2. 常見錯誤 checklist

- [ ] 沒有空洞開場
- [ ] 沒有只貼產業報告
- [ ] 沒有把問題寫成「大家都有」
- [ ] 沒有 KPI 無推導
- [ ] 沒有競品名單式分析
- [ ] 沒有功能導向而缺結果
- [ ] 沒有只列通路不證明有效
- [ ] 沒有只放甘特圖不建立信任
- [ ] 沒有財務數字無推導
- [ ] 沒有隱藏風險
- [ ] 沒有缺 Ask
- [ ] 沒有翻譯腔與空話

## 3. 簡報前檢查

- [ ] 開場 30 秒內就知道問題與代價
- [ ] 2 分鐘內講清楚方案與差異化
- [ ] 5 分鐘內講清楚數字邏輯
- [ ] 結尾有清楚 Ask 與下一步

## 4. 10 分鐘提案結構

- 開場鉤子：2 分鐘
- 問題與方案：3 分鐘
- 數字與證據：2 分鐘
- Ask 與下一步：3 分鐘

## 5. 何時不能交付

以下任一成立時，不得交付：

- Ask 還是模糊的
- 數字無法被追問
- 風險完全沒寫
- 決策者看不出為什麼現在要做
