# Dimension Coding Rules

## Theory Link

完整理論概念、動機類型意涵與研究應用請先讀：

- `references/00-theory-foundation.md`

本文件只保留標註時需要的 operational 規則與判準，避免重複理論敘述。

## Allowed Taxonomy

- `family`: `purchase_motivation`
- `subtheory`:
  - `functional`
  - `security`
  - `relational`

## Dimension Definitions

- `functional`:
  - 標註焦點是產品功能與性能驅動（如效能、品質、適配性、安裝可行性）。
- `security`:
  - 標註焦點是交易安全與可靠性驅動（如包裝完整、配送穩定、描述一致）。
- `relational`:
  - 標註焦點是互動與支持品質驅動（如溝通效率、客服支持、問題解決速度）。

## Coding Procedure

1. 逐項證據擷取動機訊號語句。
2. 每句可標註 0-2 個 subtheory，避免泛化全標。
3. 保留原句 quote，並附一句理由說明判斷依據。
4. 無法判定時標記 `insufficient`。

## Evidence Rules

- `quote` 需可被原始證據內容逐字對回。
- `reason` 要指出是「驅動因素」，不是一般心得敘述。
- `sentiment` 允許：`positive | neutral | negative`。
