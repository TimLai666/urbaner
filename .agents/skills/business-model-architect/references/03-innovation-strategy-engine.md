# Innovation Strategy Engine

## Objective

在填九要素前先建立差異化路徑，避免產出「可行但平庸」的商業模式。  
此外需把 `current_wt_state` 轉成可執行的 `wt_to_so_transition`，確保策略升維可驗證。

---

## Module 0: Epicenter Selection（震央選擇）

> 來源：Osterwalder et al. (2010)；修改自尤傳莉譯（2012）

商業模式設計應從一個**震央**出發，而非同時填九格。  
震央是設計的起點，決定哪些要素需要深度設計、哪些跟著推導、哪些可以暫緩。

### 四種震央類型與核心邏輯

#### 1. 資源導向（Resource-Driven）
**震央格**：關鍵資源（Key Resources）、關鍵夥伴（Key Partners）  
**啟動問題**：「我們已經擁有什麼？這些東西可以驅動哪些新商業模式？」  
**適用時機**：組織已有強大的基礎建設、專利、數據、合夥關係，尋找新用途  
**推導順序**：關鍵資源 → 關鍵活動 → 價值主張 → 目標客群 → 收益模式  
**典型陷阱**：只強化現有資源而忘記問「客戶真的需要嗎？」

#### 2. 產品導向（Offer-Driven）
**震央格**：價值主張（Value Proposition）  
**啟動問題**：「我們能提供什麼新的價值？這個主張需要什麼才能成立？」  
**適用時機**：有突破性創新想法、差異化產品，需要從主張往外建構整個商模  
**推導順序**：價值主張 → 目標客群 → 通路 → 顧客關係 → 關鍵資源 → 收益模式  
**典型陷阱**：迷戀產品本身而忽略交付與獲利機制

#### 3. 顧客導向（Customer-Driven）
**震央格**：目標客群（Customer Segments）、顧客關係（Customer Relationships）  
**啟動問題**：「目標客群真正需要什麼？他們現在怎麼解決？我們能讓他們更方便嗎？」  
**適用時機**：明確鎖定特定族群、市場研究已有深度洞察、服務設計為主  
**推導順序**：目標客群痛點 → 服務易取得性/便利性 → 通路 → 價值主張 → 成本  
**典型陷阱**：太靠近客戶需求而喪失商業可行性

#### 4. 財務導向（Finance-Driven）
**震央格**：收益模式（Revenue Model）、成本結構（Cost Structure）  
**啟動問題**：「有沒有新的賺錢方式？能用更低成本提供同等價值嗎？」  
**適用時機**：現有商模財務壓力大、需要轉型收費模式、成本結構重組  
**推導順序**：收益流設計 / 成本重構 → 關鍵資源 → 價值主張 → 目標客群  
**典型陷阱**：財務模型設計美觀，但客戶端需求未驗證

### 多點導向（Multi-Epicenter）
當多個震央同時啟動且互相驅動時，才考慮多點導向。  
**警告**：多點導向不是「九格都重要」的藉口——即使多點，也必須排出主震央與次震央。  
**建議**：初期商模設計優先選單一震央，多點導向保留給已有驗證基礎的商模迭代。

### 震央推薦邏輯

| 情況 | 推薦震央 |
|------|---------|
| 有強大現有資產/合夥，缺市場方向 | 資源導向 |
| 有清晰的創新想法/技術，未建商模 | 產品導向 |
| 有深度客群洞察，需要設計服務 | 顧客導向 |
| 現有商模財務不健康，需要重構收益 | 財務導向 |
| 創業初期，資源有限，需快速驗證 | 顧客導向 或 產品導向 |

## Internal Lens: Seven Faces

用七種角色視角檢查策略盲點（僅內部推演，不輸出成獨立章節）:
- 資深高階主管：產業重構與資源重配
- 內部創業家：新技術與新商模結合
- 創業家：未被滿足需求的切入設計
- 投資者：商模競爭力與資本效率
- 顧問：質疑既有假設與重構路徑
- 設計師：產品創新與商模一致性
- 社企創業家：社會與經濟價值並進

## Engine Modules

### Module A: Counterfactual Reframe

反事實問題:
- 如果不能用現有主流商業模式，怎麼交付同等價值？
- 如果目標客群不是既有主力，第二高潛力客群會是誰？
- 如果價格必須降低 50%，商模如何重構仍可獲利？

產出:
- 新假設清單
- 被挑戰的既有假設
- 可行替代路徑

### Module B: Differentiation Wedge

尋找一個「小切口、大影響」差異化楔子:
- 更快達成結果（速度）
- 更低總成本（成本）
- 更低風險（信任）
- 更高可轉移性（生態）

產出:
- 楔子定義
- 目標客群匹配度
- 防禦性來源（資料、流程、品牌、網絡效應）

### Module C: Competitive Substitution Map

定義你要取代的是:
- 直接競品
- 間接替代方案
- 客戶現行手動流程

產出:
- 取代路徑
- 轉換障礙
- 轉換誘因

### Module D: WT-to-SO Transition

把弱勢與威脅轉為強勢與機會:
- `wt_diagnosis`: 明確列出 1-3 個關鍵 WT 壓力點
- `business_model_adjustments`: 對應九要素與四構面的調整
- `so_target_state`: 轉換後的目標優勢與機會位置
- `signal_milestones`: 30/60/90 天訊號里程碑

輸出限制:
- 每個 WT 壓力點必須至少對應 1 個商模調整動作。
- 每個調整動作必須至少對應 1 個驗證指標。

## Differentiation Strategy Output (Minimum 3)

每個策略必含:
- `strategy_name`
- `core_hypothesis`
- `mechanism`
- `defensibility`
- `key_risks`
- `first_test`

## Prioritization Rubric

以 1-5 分評估每個策略:
- `novelty`
- `feasibility`
- `defensibility`
- `time_to_signal`

優先選擇:
- 高可行 + 高防禦 + 快速可驗證 的策略組合。
