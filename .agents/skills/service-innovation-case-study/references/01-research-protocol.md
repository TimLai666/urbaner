# 資料查核標準與來源驗證流程

## 查核原則

### 原則一：先查後寫，不得倒置
不允許先寫內容再找來源補充。必須先查詢、確認資料存在，再寫進報告。若查詢失敗，明確說明「查無公開資料」。

### 原則二：同名公司辨別
許多公司名稱相同但業務不同（例如：doji.com 是 AI 時尚試衣，doji.co.uk 是二手手機交易平台）。啟動前必須確認研究目標的唯一識別符（官網網址、公司全名、主要業務描述）。

### 原則三：來源可信度排序
從高到低：
1. 公司官方聲明（官網、創辦人 LinkedIn/X 發文）
2. 頂級科技媒體（TechCrunch、Business Insider、Wired、The Verge）
3. 產業媒體（RTIH、Retail Dive、Fashiontech 專門媒體）
4. 投資人/機構資料庫（Crunchbase、PitchBook、Tracxn）
5. 一般媒體報導
6. 用戶評論（App Store、TrustPilot、Reddit）——有效但需注意代表性
7. 競品/第三方評測——有效但注意利益衝突

### 原則四：來源衝突處理
當不同來源對同一事實有不同說法（如：公司成立時間、融資金額），處理方式：
1. 優先採用可信度最高的來源
2. 在報告中明確標注衝突：「不同來源有出入，以 [來源] 為準，原因是 [理由]」
3. 不允許任意選一個來源而不說明

## 必查清單（每個新案例必須執行）

### A 類：公司基礎資料
- [ ] 官方網站（確認業務描述）
- [ ] Crunchbase / PitchBook / Tracxn（融資資料、成立時間、員工數）
- [ ] LinkedIn Company Page（員工數、地點、成立時間）
- [ ] TechCrunch / 主流媒體最新報導（最新動態）

### B 類：創辦人背景
- [ ] 每個創辦人的 LinkedIn（前雇主、學歷）
- [ ] 創辦人個人網站或個人介紹頁（補充細節）
- [ ] Google Scholar（若有學術背景）
- [ ] 媒體訪談（引言與創辦故事）

### C 類：投資背景
- [ ] 每個投資方的官網或介紹頁（確認投資方性質）
- [ ] 投資新聞稿或媒體報導（確認是否真的投了）
- [ ] 天使投資人的公開聲明（確認參與）
- [ ] **必須主動查證「是否確定這個投資方有投資這家公司」**，不能只根據一則報導就斷言

### D 類：產品與功能
- [ ] App Store 或 Google Play 頁面（功能說明、用戶評論、更新紀錄）
- [ ] 官網功能介紹頁
- [ ] 媒體產品評測（至少 2-3 個不同來源）

### E 類：市場數據
- [ ] 市場規模與成長率（研究機構報告：Research and Markets、Statista、Precedence Research）
- [ ] 退貨率、用戶行為等行業數據（NRF、CNBC、行業協會）
- [ ] 競爭者清單（Tracxn 競爭者列表）

### F 類：外部評價
- [ ] App Store 用戶評論（正負面各挑 2-3 則具代表性的）
- [ ] 競品評測文章（注意撰寫者是否有利益衝突）
- [ ] Vogue Business、WWD 等時尚媒體（如有報導）

## 來源標注格式

### 在報告正文中
單一引用：（來源：TechCrunch 2025/05/15）
多個來源：（來源：Business Insider 2025/03；Benzinga 2025/03）

### 在時間線節點後
> 來源：[媒體名稱]（[日期]）— "[報導標題的直接引文摘要]"

### 在 PESTEL 因素中
來源依據：[完整來源說明，含報導內容的摘要說明]

### 在參考資料區段
統一格式：
```
[編號]. **[媒體/來源名]**（[日期]）— [標題或說明]
   [URL]
```

## 特殊情況處理

### 情況一：找不到某項數據
寫法：「[X] 尚未公開揭露任何數據，無可查核的公開來源。」
**不允許**：假設數字、用「大約」引用未確認的數字

### 情況二：數據過時
若唯一找到的來源已超過 1 年，標注：「最近可得數據為 [日期]，目前狀況可能已有變化。」

### 情況三：媒體引言的間接引用
若引言是媒體的報導而非原始來源（如媒體引用了創辦人的 LinkedIn 發文），標注原始來源：
「Dargan 在 LinkedIn 表示（引自 TechCrunch 2025/05/15）：...」
盡量找原始來源（直接引創辦人的 LinkedIn / X），不只依賴媒體轉述。
