# Dimension Coding Rules

## Theory Link

完整理論概念、口碑動機類型與研究應用請先讀：

- `references/00-theory-foundation.md`

本文件只保留標註時需要的 operational 規則與判準，避免重複理論內容。

## Allowed Taxonomy

- `family`: `wom_motivation`
- `subtheory`:
  - `altruistic`
  - `social_identity`
  - `self_enhancement`
  - `emotional_expression`

## Dimension Definitions

- `altruistic`:
  - 標註焦點是「為幫助他人」的分享目的與風險降低訊號。
- `social_identity`:
  - 標註焦點是社群互動、認同與歸屬訊號。
- `self_enhancement`:
  - 標註焦點是能力、專業、經驗、成就展現訊號。
- `emotional_expression`:
  - 標註焦點是情緒宣洩或態度表達，常伴隨強烈正負向語氣。

## Coding Procedure

1. 逐項證據找出分享意圖語句。
2. 每句最多標 2 類動機，避免過度重疊。
3. 每則標註都要附原句與判斷理由。
4. 沒有明顯分享意圖時，標示 `insufficient`。

## Disambiguation Rules

- `social_identity` vs `self_enhancement`：
  - 若重點在「被社群看見/互動」，優先 `social_identity`。
  - 若重點在「展示我很懂/我做到了」，優先 `self_enhancement`。
- `altruistic` 需有明確「幫助別人」語意，不可僅因內容詳細就標註。
