### ⚡ 三大核心洞察（策略總覽）

**🇺🇸 US — 送禮場景 × 多功能套組是市場決策主軸**
> 最能拉開不同顧客的兩個屬性 — 「適不適合送禮給男性」與「多功能套組」 — 強度遠超其他項目。父親節 / 聖誕節送禮情境是絕大多數美國電剪購買決策的觸發點。

**🇯🇵 JP — 附件齊全度 × 0.5mm 精度 × 電源類型是日本決策三軸**
> 最能拉開不同顧客的前 3 個屬性都是「規格深度」：附件件數、梳齒款式、可調梳齒，強度比第 4 名加起來還高。日本顧客重視 38 段刻度、附件套組完整、CP 值高、**丸洗い（整支可水洗）**。

**🌐 兩市場共識 — IPX7+ × ≥7 件套組 = 2026 新品最低門檻**
> 兩市場的顧客分群分析與社群觀察結論一致；新 SKU 開發必須踩上這道規格線。

## 🎯 第三頁：STP 策略分析

> STP 三步驟 — 先看「誰在買」（分群）、再決定「要服務誰」（鎖定優先族群）、最後想清楚「我要是誰」（在競爭群裡的位置）。美日決策軸完全不同 — 美國由「送禮場景」驅動、日本由「規格深度」驅動。

### 🎯 顧客分群 — 美日各有 3 種顧客

（內容同雙市場頁的 Segmentation 表，每張卡左側 5px 市場色 accent）

**🇺🇸 美國 3 個顧客群**

  - 類別組成：Beard 87.5% / Nose-Ear 6.8% / Body 5.6%
  - 他們是誰：為男士挑禮：在意送禮場景、性別感、附件齊全度
- **S2：日常自用大眾**（n=3,337, 76.5%, ★3.18）
  - 類別組成：Beard 94.8% / Nose-Ear 4.2% / Body 1.0%
  - 他們是誰：買來自己每天用：在意鬍鬚與耳鼻、耐用、可充電
- **S3：USB-C 高端鐵粉**（n=173, 4.0%, ★4.48）
  - 類別組成：Beard 75.7% / Body 17.9% / Nose-Ear 6.4%

**🇯🇵 日本 3 個顧客群**
- **S1：CP 值優先大眾**（n=6,559, 91.6%, ★3.46）
  - 類別組成：Nose-Ear 59.4% / Beard 40.6%
  - 他們是誰：節儉派：愛乾電池款、鼻毛 + 鬍鬚兩用、看重耐用易用
- **S2：鬍鬚講究客**（n=561, 7.8%, ★4.01）
  - 類別組成：Beard 75.2% / Nose-Ear 24.8%
  - 他們是誰：深度玩家：38 段精度、多附件、機身大小錙銖必較

**🔬 統計方法**：非監督式分群：每則評論的 114 維「屬性顯著度向量」→ StandardScaler 標準化 → PCA 降至保留 85% 變異 → K-Means（K=3-6 掃描，以 Silhouette + 最小群體 ≥ 5% 護欄選 K=3）→ 用 ANOVA 找各 cluster 最具區隔力的屬性 → 命名 persona。

**📊 數值解讀**：人數 n = 該 cluster 包含的評論則數；占比 = n／市場總評論。Avg★ = 該 cluster 內所有評論的星等平均，用顏色標示滿意度：≥4.0 綠 / ≥3.5 橙 / <3.5 紅。「他們是誰」由 cluster 內 Top 5 顯著屬性人工解讀後命名。

**解讀**：
- **美日大眾族群動機差很多**：US 大眾「日常自用」買來自己每天用 — 鬍鬚 + 耳鼻一機通用；JP 大眾「CP 值優先」愛乾電池款，看重耐用與易用。
- **高滿意小族群才是利潤池**：US S3 USB-C 高端鐵粉（★4.48）與 JP S2 鬍鬚講究客（★4.01）— 雖只佔 4% / 7.8%，但滿意度遠高於大眾，且願意為精度與充電付溢價。

### 📈 鎖定優先族群 — 哪些屬性最能拉開不同顧客的差距？

（圖表同雙市場頁的 ANOVA Top 10）

**🔬 統計方法**：One-way ANOVA — 以 K-Means 分出的 cluster 為因子、114 個屬性的顯著度分數為依變數，逐項檢定。同雙市場頁的 ANOVA，但此處作為「Targeting 優先序」的依據。

**📊 數值解讀**：F 值大 = 該屬性是「能拉開不同顧客群」的關鍵訊號 — 做行銷時應針對 Top 屬性下文案、做產品時應針對 Top 屬性提升規格。p < 0.001 為高度顯著（本表 Top 10 屬性全為 p < 0.001）。

### 🔥 哪個 SKU 在哪個屬性表現最好？— 屬性 × SKU 口碑熱圖

**圖檔**：US / JP 各一張熱圖，Top 20 屬性 × Top 12 SKU 的品質分網格

## ⚖️ 第四頁：Conjoint 偏好分析

> 從 11,523 則 Amazon 評論還原顧客「真正在比什麼」，再用真實售價回推「願意為品質多付多少錢」。三張圖一起看：消費者在意什麼 → 願意多付多少 → 把規格做上去市佔能跳多少。

### ⚖️ 消費者最在意哪些屬性？

（屬性重要性表同執行頁）

**🔬 統計方法**：Revealed-Preference Logistic Conjoint（觀察型偏好聯合分析）— 以「avg★ ≥ 該市場平均」為被解釋變數，將每項屬性切成 Low/Mid/High 三層做 dummy encoding，逐屬性 Split-model Logistic Regression 估 part-worth。

**📊 數值解讀**：屬性重要性 (%) = 該屬性的 part-worth 全範圍（max − min） ÷ 所有屬性 part-worth 全範圍總和，再化為百分比。Conjoint 是行銷研究經典方法，數字越大代表此屬性對「買 / 不買」決策的拉扯力越大。

**解讀**：
- **美日決策軸完全不同**：US 一根「功能合一數」就吃掉 51.5% 重要性 — 套組組合是壓倒性決策因子；JP 前 6 個屬性都在 10% 以上、分布均勻，代表日本顧客同時看附件數、長度段數、精度、續航。
- **產品溝通方向**：US Listing 第一張主圖必須講清楚「幾合一」；JP Listing 第一條 bullet 要把 X 個附件 / X 段 / X mm 全部列出。

## 📐 第八頁：統計方法論

> 把儀表板用到的每一個統計／量化方法都攤開來：學術定義、本專案怎麼使用、出現在哪些卡片。適合做為產學報告、口頭簡報、或交給數據工程師重現分析時的參考。

### 🧩 K-Means Clustering（K 平均分群）

**📚 學術定義**
非監督式分群演算法。給定 n 個觀察值與目標群數 k，將觀察值分到 k 個 cluster 使每個觀察值都屬於距離最近的 cluster centroid（質心）。演算法以隨機 / k-means++ 初始化質心，反覆執行「分配 → 更新質心」步驟，目標是最小化群內平方誤差（Within-Cluster Sum of Squares, WCSS）：

```
argmin Σᵢ Σ_x∈Cᵢ ‖x − μᵢ‖²
```

屬於硬性分群（hard clustering），每個觀察值剛好屬於一個 cluster。

**🔧 在此專案如何使用**
用於 **STP Segmentation**。每則 Amazon 評論視為一個觀察值，其 114 維「屬性顯著度分數」（Axis A，0–7）為特徵向量。為避免維度災難與重疊雜訊，先以 StandardScaler 標準化、PCA 降至保留 85% 變異後，再執行 K-Means。K 值掃描 K ∈ {3, 4, 5, 6}，採用「Silhouette 最大 + 最小群體 ≥ 5%」雙條件選擇，兩市場最終 K = 3。

**🎯 出現在儀表板**：雙市場對比 / 區隔組成 ｜ 雙市場對比 / 樹狀圖 ｜ STP / 顧客分群

---

### 📐 PCA — Principal Component Analysis（主成分分析）

**📚 學術定義**
線性降維方法。透過 SVD（奇異值分解）或共變異數矩陣的特徵分解，找出資料中變異最大的正交方向（principal components）。第一主成分 PC1 解釋最多變異，PC2 在與 PC1 正交的子空間內解釋次多變異，依此類推。保留前 k 個主成分即可在保留大部分變異的前提下將維度從 d 降到 k。

**🔧 在此專案如何使用**
兩個用途：
1. **K-Means 前的降維**：把 114 維屬性向量降到保留 85% 變異的主成分空間後再分群，改善 K-Means 在高維下的不穩定。
2. **市場定位地圖（Perceptual Map）**：將每支 SKU 的 114 維品質向量（Axis B，0–10）投影到 PC1 × PC2 平面，視覺化各 SKU 的相對位置。PC1 通常代表「整體品質強度」，PC2 代表「品質維度間的偏好取捨」。

**🎯 出現在儀表板**：STP / 顧客分群（前處理）｜ STP / 市場定位地圖 ｜ 雙市場對比 / 樹狀圖

### 🪞 Silhouette Score（輪廓係數）

**📚 學術定義**
用於評估分群品質的指標，範圍 [-1, +1]，越接近 1 越好。對每個觀察值 i：

```
s(i) = (b(i) − a(i)) / max(a(i), b(i))
```

其中 a(i) = 與「同 cluster 其他點」的平均距離；b(i) = 與「最近其他 cluster 中點」的平均距離。整體 silhouette score = 所有點 s(i) 的平均。

**🔧 在此專案如何使用**
用於 K-Means 的 K 值選擇 + 分群品質報告。兩市場結果：**US silhouette = 0.358**（中等品質，分群邊界有模糊重疊）；**JP silhouette = 0.570**（佳，分群清晰分明）。JP 較高是因為日本最大族群「CP 值優先大眾」高度集中於同質決策軸（電源類型 + 鼻毛），與其他族群差異明顯。

**🎯 出現在儀表板**：雙市場對比 / 樹狀圖（標註於方法註記）｜ STP / 顧客分群

### 📊 One-way ANOVA（單因子變異數分析）

**📚 學術定義**
檢定「≥ 3 個獨立群體的平均值是否有顯著差異」。虛無假設 H₀：μ₁ = μ₂ = … = μₖ；對立假設 H₁：至少有兩個群體均值不同。

F 統計量：
```
F = MS_between / MS_within
```

其中 MS_between = 組間變異 / (k − 1)，MS_within = 組內變異 / (N − k)。F 服從 F-分配；p < 0.05 視為統計顯著。F 越大代表組間差異相對組內變異越大。

**🔧 在此專案如何使用**
用於 **STP Targeting**。對 114 個屬性逐一執行 ANOVA：因子 = K-Means 分出的 cluster（3 群），依變數 = 該屬性的評論顯著度分數。F 越大的屬性 = 越能「拉開不同顧客群差距」的關鍵屬性。

兩市場 Top 屬性：
- US Top 1 = `gift_suitability_men`（F = 1299, p ≈ 0）
- JP Top 1 = `total_attachments_count`（F = 2630, p ≈ 0）

**🎯 出現在儀表板**：雙市場對比 / 屬性區隔力 ｜ STP / 鎖定優先族群

### ⚖️ Conjoint Analysis — Split-Logit Part-Worth

**📚 學術定義**
Conjoint 是行銷研究經典方法，用於分解「複雜產品偏好」為「各屬性 × 各水準的部分效用 (part-worth)」。Split-Logit 是其中一種估計方式 — 對「avg★ ≥ 市場平均 → 偏好」做 logistic regression，每個屬性切成 Low/Mid/High 三層用 dummy variables encoding，逐屬性估 part-worth。

屬性重要性 (%) = max(part-worth) − min(part-worth)，再除以所有屬性 part-worth 全範圍總和。

**🔧 在此專案如何使用**
**Revealed-Preference Conjoint**（觀察型）— 不做問卷，直接以 Amazon 真實 SKU 為實驗卡片：avg★ ≥ 市場平均 → preferred=1。估出每個屬性對「買 / 不買」決策的拉扯力 (part-worth)。

樣本：US n=52 SKUs / JP n=36 SKUs。兩市場差異戲劇性：US 「功能合一數」單項佔 51.5%、JP 前 6 個屬性都在 10% 以上分布均勻。

**🎯 出現在儀表板**：執行儀表板 / 屬性重要性 ｜ Conjoint / 消費者最在意哪些屬性

### 💰 Willingness-to-Pay（WTP，願付溢價）

**📚 學術定義**
貨幣度量效用（money-metric utility）。在帶價格變數的 choice-based conjoint 中，每個屬性的 part-worth 可換算為「願意多付多少錢」：

```
WTP_attribute = β_attribute / |β_price|
```

其中 β_attribute = 該屬性 part-worth、β_price = 價格變數的 logistic regression 係數。假設 β_price < 0（價格越高購買機率越低），WTP 才有正確的經濟學意義。

**🔧 在此專案如何使用**
在 Conjoint 模型加入真實 Amazon 售價作為連續變數，估出 β_price 後計算 WTP。

- **樣本**：US 19 個 SKU 有完整售價 / JP 21 個。
- **結果**：US β_price = -0.0212（符合假設，可解讀）；**JP β_price = +0.0004（違反假設，僅供方向參考）** — 原因是日本市場高價與高品質正相關（高價＝高品質訊號），售價無法分離出純粹的價格敏感度。

**🎯 出現在儀表板**：Conjoint / 為品質升級願意多付多少 ｜ Conjoint / WTP 完整對照表

### 🎯 MNL — Multinomial Logit Share-of-Preference

**📚 學術定義**
離散選擇模型，源於 McFadden（1974）的隨機效用理論（Random Utility Theory）。假設消費者效用 Uᵢ = Vᵢ + εᵢ，其中 εᵢ 服從獨立同分布的 Gumbel 分配。選擇 SKU i 的機率：

```
P(i) = exp(Vᵢ) / Σⱼ exp(Vⱼ)
```

其中 Vᵢ 是觀察到的代表性效用（observed deterministic utility），由 Conjoint 估出的 part-worth × 該 SKU 的屬性值加總而成。

**🔧 在此專案如何使用**
兩個用途：
1. **市佔位置**（執行頁 + 最優產品頁）：選擇集 = URBANER 自家 + 競品 + 24 張設計卡，計算 URBANER 在競爭環境中的相對偏好份額。結果：US URBANER 僅 5.21% / JP 11.26%。
2. **升級模擬**（Conjoint 頁）：將 URBANER 在某屬性的品質設為 10（理想值），其餘維持，重算 P(URBANER)。增量 = 該屬性升級的邊際市佔回報。

**🎯 出現在儀表板**：執行 / 兩市場目前份額位置 ｜ 最優產品 / 在競品環伺下 ｜ Conjoint / 升級模擬

### 🎯 Ideal-Point RMS Distance（理想點均方根距離）

**📚 學術定義**
多屬性偏好模型中，「理想點 (ideal point)」是代表「完美品質」的向量（在本專案為 [10, 10, …, 10]，因 Axis B 品質分上限為 10）。每支 SKU 與理想點的歐式距離可作為「離顧客理想多遠」的代理指標。為了避免高維下距離數值膨脹，採 RMS（Root Mean Square，均方根）版本：

```
d = √(mean((qᵢ − 10)²))
```

其中 qᵢ 是該 SKU 在第 i 個屬性的品質分數。

**🔧 在此專案如何使用**
用於 **STP Positioning**：在 84 個追蹤 SKU 內計算每支 SKU 的 RMS 距離。

**URBANER 自家旗艦**（從 7 個確認 URBANER + 15 個無品牌訊號 ASIN 中取最小距離）：
- US：B0GL2DKVQH，d = 2.49
- JP：B07CYZH2XC，d = 3.43

**全市場頂尖**（全部 SKU 含競品）：
- US：B0FL267TCG（Ufree），d = 1.60
- JP：B0GBWZBMS5，d = 1.97

**🎯 出現在儀表板**：雙市場對比 / URBANER 自家旗艦 vs 市場頂尖

### 🏆 Conjoint Utility Maximization + Sigmoid 正規化

**📚 學術定義**
在已估出 part-worth 後，可對「設計卡片」（未上市的假設產品組合）計算總效用：

```
U_card = Σ_attr part_worth(attr, level)
```

為了把效用轉成 0–1 的「預估購買率」，先做 Z-score 正規化避免 sigmoid 飽和，再套 logistic 函數：

```
P(購買) = 1 / (1 + exp(−U_norm))
```

**🔧 在此專案如何使用**
用於 **最優產品組合** 卡片頁。以 7 個屬性 × 3 個水準的全因子設計產生 24 張產品卡，計算每張的 P(購買)。

兩市場最佳組合（皆為 Card 22）：
- US: P(購買) = 83.3%（USB-C × 38 段 × ≥10 件套組）
- JP: P(購買) = 89.1%（IPX7 × 90 分 × 1mm 精度）

**🎯 出現在儀表板**：最優產品組合 / 兩市場最佳產品組合

### 🏷️ Hedonic Pricing Regression（屬性定價回歸）

**📚 學術定義**
經濟學方法，將商品價格分解為各屬性的隱含價值。以 ln(price) 為因變數、屬性品質分為自變數做 OLS 回歸：

```
ln(Pricej) = β₀ + Σ βi × Qualityij + εj
```

半對數模型下，βᵢ × 100% ≈ 該屬性品質提升 1 個單位時，市場價格的百分比變化。與 WTP 不同：Hedonic 反映「市場上競品實際的定價邏輯」，WTP 反映「消費者主觀願付」。

**🔧 在此專案如何使用**
**延伸分析路線 C**（extended_analysis_report.md），可作為 WTP 的對照參考。結果樣本太小（US 19 / JP 21）且共線性高，主要用於檢查「市場定價是否與消費者偏好一致」。目前儀表板未直接顯示此分析（在 docx 報告內），未來如要納入可開新卡片。

**🎯 出現在儀表板**：（僅 docx 報告，儀表板暫未顯示）

### 📝 Qualitative Content Analysis（質性內容分析）

**📚 學術定義**
系統性閱讀文本資料、依研究問題建立編碼框架（coding scheme），再對每段內容指派類別。頻率計數（frequency coding）給出各類別的相對強度。與量化分析不同：不做統計推論，而是「主題的存在、強弱、覆蓋廣度」的歸納整理。

**🔧 在此專案如何使用**
用於 **社群洞察**。對 social_us/insights.md 與 social_jp/insights.md 兩份合成自 Reddit、Badger&Blade、Sharpologist、知恵袋、価格.com、マイベスト、note、5ch 等 20+ 平台的社群觀察，按 9 個產品類別 × 痛點 / 好評 / 競品偏好編碼。再對品牌做頻率計數：例如 Panasonic 出現於 9 個類別、Philips Norelco 出現於 5 個類別。

**🎯 出現在儀表板**：社群洞察 / 跨類別共通洞察 ｜ 社群洞察 / 9 類別深度 ｜ 社群洞察 / 社群被提及的主要競品

### 🪢 Axis A × Axis B 雙軸評分系統

**📚 學術定義**
本專案自定義的評論評分框架：

- **Axis A — 顯著度（Salience，0–7）**：每則評論 × 每個屬性的「關鍵字密度 × 內容長度加權」分數。0 = 該屬性未出現在此評論；7 = 該屬性是評論的中心議題。
- **Axis B — 品質（Quality，0–10）**：每商品 × 每個屬性的「星級基線 + 屬性負向關鍵字扣分 + 整體情緒微調」。由該商品所有評論彙總而得，反映此商品在該屬性上的市場觀感。

**🔧 在此專案如何使用**
STP 分群以 Axis A 為特徵向量（看「評論在意什麼」），Conjoint 與 Positioning 以 Axis B 為特徵向量（看「商品做得怎樣」）。兩軸分工讓「需求側（評論顯著度）」與「供給側（商品品質）」分析得以對接。

**🎯 出現在儀表板**：所有 STP / Conjoint / Positioning 分析皆使用此底層資料
